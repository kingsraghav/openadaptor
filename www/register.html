Basics

What is registration

Registration is a process for keeping record of all instances of Openadaptor used in an organisation. When an adaptor is started it sends information about its environment, such as OA version, Java version, operating system name and more. That information is stored in a central repository.

Environment

Registration server

Registration server is required to collect data sent by disparate adaptors. A sample adaptor has been written that exposes a Webservice reader and a database writer. The adaptor's definition is in: example\spring\register\oa-register-server.xml. The adaptor connects to a database where it expects a schema defined in sql\register\mysql\schema.sql (tested for MySql).

Registration client

Every adaptor with enabled registration (disabled by default) will attempt to send registration data via a web service, right after it is started. Sending happens asynchronously and as such is safe - absence of the registration server will not stop the adaptor from starting or stopping. Registration can be enabled by providing server's URL in registration.properties.
